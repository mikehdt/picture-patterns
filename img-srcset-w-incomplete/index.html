<img> with srcset -w syntax, no source attribute

By deliberately leaving off the otherwise requires sources attribute, we often get some weird browser behaviour. You should not leave off the sources attribute, this is just to provide a counterpoint as to why it is needed.

The first image has the 320w but is still missing sizes.

The second image leaves off the 320w and is missing the sizes attribute.

The third image uses -x syntax as some weird behaviours can occur when both syntax types are on the same page.

Source:


					<img src="img-320.png"
					     srcset="img-1920.png 1920w,
					             img-1280.png 1280w,
					             img-960.png   960w,
					             img-640.png   640w,
					             img-480.png   480w,
					             img-320.png   320w">

					<img src="img-320.png"
					     srcset="img-1920.png 1920w,
					             img-1280.png 1280w,
					             img-960.png   960w,
					             img-640.png   640w,
					             img-480.png   480w,
					             img-320.png">

					<img src="img-320.png"
					     srcset="img-320.png 1x,
					             img-640.png 2x">
